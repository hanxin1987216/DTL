strate.shouldRun()判断的是当前时刻是否满足 DTimerStrategy中的日期(_mode,_date)和起止时间(_startTime,_endTime) runable判断的是在满足strate.shouldRun()的条件后task时task是否可以运行 解决一次性任务和循环执行任务的问题

    if (_task != NULL) {
        return;
    }
    _task = const_cast<ITimerTask*>(&task);
    _canceled = false;
    _task->setValid(true);
    ITimerTask *tsafe = _task;//use this poiter for thread safty
    //DTimerStrategy *tstrate = const_cast<DTimerStrategy*>(&strate);

    bool runable = true;
    //while (!_canceled) {
    while (tsafe->isValid()) {
        wait(1);
        if (strate.shouldRun()) {
            if (runable) {
                if (strate._endTime == DTimerStrategy::INVALID_TIME) {
                    //_endTime无效，task只运行一次
run_once:
                    tsafe->run();
                    runable = false;
                } else {
                    //周期为0时task只运行一次
                    if (strate._interval == 0) {
                        goto run_once;
                    }
                    //每隔interval秒运行一次task
                    //while (!_canceled && strate.shouldRun()) {
                    while (tsafe->isValid() && strate.shouldRun()) {
                        tsafe->run();
                        wait(strate._interval);
                    }
                }
            } //if (runable)
        } else {
            runable = true;
        }
    }
}
void DTimer::sheduleAtFixedRate ( const ITimerTask &  task,
const Date &  firstTime,
unsigned int  interval 
)

sheduleAtFixedRate try to make the period between the begining time of tasks are exactly [interval] seconds for example,if a task start at 9:00:00,and [interval] is 60 seconds we assume task.run() will cost 30 seconds

if call shedule(), the next time call task.run() will be 9:01:30, that means 30s runtime + 60s interval

if call sheduleAtFixRate(), the next time call task.run() will be 9:01:00, the cost of task.run() is omitted but if task.run() cost 90 seconds,the next time call task.run() will delayed to 9:01:30
